**1. What `self` really is**

- When you call `rex.greet()`, inside `greet` the name `self` means **`rex`**.
- So `self.name` is `rex.name`, `self.energy` is `rex.energy`.

---

**2. Using `self` with attributes**

Inside a method, you always read the object’s data like this:

```python
def describe(self):
    return f"This pet is named {self.name} and has energy {self.energy}"
```

Key pattern: `self.attribute` → “use **my** attribute”.

---

**3. Methods depend on current state**

If you do:

```python
pet1 = Pet("rex", 93)
print(pet1.describe())   # uses energy 93

pet1.energy = 40
print(pet1.describe())   # now uses energy 40
```

The method doesn’t “remember” old values.  
Each call uses **whatever is stored on the object at that moment**.

---

**4. Mental model**

- Outside: you write `rex.greet()`.
- Python secretly turns it into: `Pet.greet(rex)`.
- Inside: `self` is just the parameter name that receives `rex`.
