**Classes and instances — summary**

- A **class** is a **blueprint** that defines what a kind of object should have and do.
    
    - Example: **Pet** (says every pet has a name, age, etc.)
- An **instance** is **one actual object** made from that blueprint.
    
    - Example: `rex` (a Pet with name = "Rex", age = 5)
    - Many instances can come from one class: `rex`, `luna`, `max` all from **Pet**.
- Python has a naming habit:
    
    - **Classes** → **CapitalCase**: `Pet`, `Dog`, `Cat`
    - **Instances / variables** → lowercase / `snake_case`: `rex`, `my_dog`, `first_pet`
- Similar idea with lists:
    
    - The _concept of a list_ = blueprint
    - `[1, 2, 3]`, `["a", "b"]` = different **list instances** made from that one idea.

**“A class is a CapitalCase blueprint (like `Pet`), and an instance is one real lowercase object made from it (like `rex`).”**
