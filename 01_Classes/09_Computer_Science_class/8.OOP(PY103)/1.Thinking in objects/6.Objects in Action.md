---

**1. Creating a Random object**

> `rng = random.Random()`  
> Calling `Random()` with parentheses **creates a new object (instance)**.

- `import random` → brings in the **module** (toolbox).
- `random.Random()` → makes a **Random object**.
- `rng` → variable that **refers to that object**.

Think:  
**module = toolbox**,  
**object = your personal gadget built from that toolbox**.

---

**2. Object, methods, and data**

Using the object:

```python
number = rng.randint(1, 10)
```

Breakdown:

- `rng` → **object** (Random instance).
- `randint` → **method** (action the object can perform).
- `1, 10` → **arguments** (input data you pass into the method).
- `number` → **returned data** (a normal `int`, not an object).

Flow to remember:

> **object → call method → get data**

---

**3. Another method example: `choice`**

```python
pets = ["cat", "dog", "lizard"]
picked = rng.choice(pets)
```

Roles:

- `rng` → **object**
- `choice` → **method** on the object
- `pets` → **input data** to the method
- `picked` → **output data** (one element from `pets`)

Again, same pattern:

> **object (`rng`) → method (`choice`) → data (`picked`)**

---

**4. Why use a Random object at all?**

Instead of only using module-level functions like `random.randint(...)`, you sometimes want a **personal random generator**:

- You can create **separate streams** of randomness:
    
    ```python
    rng_game = random.Random()
    rng_tests = random.Random()
    ```
    
- You can control them using **seeds** for reproducible runs:
    
    ```python
    rng = random.Random(42)  # same sequence every run
    ```
    
- You can **pass `rng` around** to functions instead of calling global `random` everywhere, which makes big programs easier to reason about.

---

**5. Key contrasts to remember**

- `random` (module)
    
    - Comes from `import random`
    - Use like: `random.randint(1, 10)`
    - Feels like calling tools straight from a toolbox
- `rng = random.Random()` (object)
    
    - A **specific Random instance** you created
    - Use like: `rng.randint(1, 10)`, `rng.choice(items)`
    - Feels like your **own gadget** you can hand to different parts of the program

---

**6. Core takeaway (short version)**

- `random.Random()` with `()` → **creates an object instance**.
- Methods on that object (like `.randint`, `.choice`) → **actions** the object can perform.
- The values you store in variables like `n1`, `n2`, `picked` → plain **data** returned from those methods, _not_ objects.

**Random object basics**

- `import random` → load the **module** (toolbox).
- `rng = random.Random()` → create a **Random object (instance)**.
- `rng` is just a **variable** that refers to that object in memory.

**Object → method → data pattern**

Python

```python
number = rng.randint(1, 10)
```

- `rng` → **object**
- `randint` → **method** (action the object can do)
- `1, 10` → **arguments** (input data)
- `number` → **returned data** (an `int`)

> Pattern: **object → call method → get data**

**Another method example**

Python

```python
pets = ["cat", "dog", "lizard"]
picked = rng.choice(pets)
```

- `rng` → object
- `choice` → method
- `pets` → input data
- `picked` → output data (one of the pets)

**Why use a Random object instead of just `random.randint`?**

- **Separate streams** of randomness (e.g., `rng_game`, `rng_tests`).
- **Reproducible** results with seeds: `rng = random.Random(42)`.
- Easier to **pass around** a specific RNG instead of using global module functions everywhere.

**Module vs object**

- Module function: `random.randint(1, 10)`
- Object method: `rng.randint(1, 10)` after `rng = random.Random()`

Same _kind_ of result (a number), but the second uses **your own RNG object**.
