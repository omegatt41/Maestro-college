# Insertion Sorting method
	1. insertion sorting method can be faster on a nearly sorted list
		1. but as the list grows and becomes less sorted this method can be slow as it moves all numbers around and is BigO(n^2) so as the list grows the time it takes to run this method grows exponentialy with the size of the list
```python
def insertion_sort(nums):
    # sort nums in place
    for i in range(1, len(nums)):
        current = nums[i]
        j = i - 1

        #shift bigger numbers right
        while j >= 0 and nums[j] > current:
            nums[j + 1] = nums[j]
            j -= 1
        nums[j + 1] = current
    return nums

print(insertion_sort([5,2,8,4]))
```

**Insertion sort core idea**

Insertion sort builds a sorted section on the left.

Each step:

1. Take the next unsorted value.
2. Compare it with values on the left.
3. Shift bigger values one spot right.
4. Insert the current value into the open spot.

**Key Python pattern**

Python

```python
def insertion_sort(nums):
    for i in range(1, len(nums)):
        current = nums[i]
        j = i - 1

        while j >= 0 and nums[j] > current:
            nums[j + 1] = nums[j]
            j -= 1

        nums[j + 1] = current

    return nums
```

**Important lines**

Python

```python
nums[j + 1] = nums[j]
```

Shifts a bigger number right.

Python

```python
nums[j + 1] = current
```

Places the current value into the correct slot.

**Time complexity**

Average case:

Text

```text
O(n²)
```

Why: each item may need to scan/shift across many earlier items.

Best case:

Text

```text
O(n)
```

Why: if the list is already or nearly sorted, the `while` loop barely runs.

**One-sentence summary**

Insertion sort repeatedly takes the next value and inserts it into the correct position in the already-sorted left side.
