def bubble_sort_with_counts(original_list):
    """
    Sorts a copy of the list using Bubble Sort.
    Tracks and returns the total number of swaps.
    """
    # Create a new list copy to avoid modifying the original list
    arr = list(original_list)
    n = len(arr)
    swap_count = 0
    # Traverse through all array elements
    for i in range(n):
        # Last i elements are already in place
        for j in range(0, n - i - 1):
            # Swap if the element found is greater than the next element
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swap_count += 1
    return arr, swap_count
  
def insertion_sort_with_counts(original_list):
    """
    Sorts a copy of the list using Insertion Sort.
    Tracks and returns the total number of shifts (element movements).
    """
    # Create a new list copy to avoid modifying the original list
    arr = list(original_list)
    shift_count = 0
    # Traverse from 1 to len(arr)
    for i in range(1, len(arr)):
        key = arr[i]
        # Move elements of arr[0..i-1] that are greater than key
        # to one position ahead of their current position
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            shift_count += 1
            j -= 1
        # Place the key into its correct position
        arr[j + 1] = key
    return arr, shift_count

# Notes on Comparison:
**Bubble Sort vs. Insertion Sort**

**Bubble sort**

- Compares **neighboring values**.
- If the left value is bigger than the right value, it **swaps** them.
- Large values “bubble” toward the right side.
- Needs repeated passes until the list is sorted.

Example:

Python

```python
[5, 2, 4]
```

First comparison:

Python

```python
5 > 2
```

So bubble sort swaps:

Python

```python
[2, 5, 4]
```

Then it compares `5` and `4`, and swaps again:

Python

```python
[2, 4, 5]
```

**Insertion sort**

- Builds a **sorted left section**.
- Takes the next value and inserts it where it belongs.
- Bigger values shift right to make room.
- Often works well when the list is already almost sorted.

Example:

Python

```python
[2, 5, 8, 4]
```

The left side `[2, 5, 8]` is already sorted.  
To insert `4`, shift `8` and `5` right:

Python

```python
[2, 5, 8, 8]
[2, 5, 5, 8]
```

Then insert `4`:

Python

```python
[2, 4, 5, 8]
```

**Main difference**

|Algorithm|How it scans|How it moves values|
|---|---|---|
|Bubble sort|Checks neighboring pairs|Swaps neighbors|
|Insertion sort|Looks left through the sorted section|Shifts bigger values right, then inserts|

**When one may be preferable**

Use **insertion sort** when the list is **nearly sorted**.

Example:

Python

```python
[1, 2, 3, 5, 4]
```

Only `4` is slightly out of place.  
Insertion sort only needs to shift `5` right and insert `4`.

Bubble sort can also fix it, but insertion sort is often cleaner for nearly sorted data.

**Complexity**

For large lists:

Python

```python
Bubble sort:    O(n²)
Insertion sort: O(n²)
```

That means both can become slow as the list gets big.

Important note:

- Insertion sort can be faster on **already sorted** or **nearly sorted** lists.
- But for large messy lists, both have similar time complexity.
