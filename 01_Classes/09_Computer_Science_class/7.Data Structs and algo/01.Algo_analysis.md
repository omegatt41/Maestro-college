**Lesson: Welcome to algorithms**

**Core idea:**

> An **algorithm** is a clear set of steps for solving a problem.

It does not have to be complicated, and it does not only mean “code.”  
Everyday examples can be algorithms too, like making a sandwich, following a recipe, or helping customers in line.

**Data structures vs. algorithms**

|Concept|Meaning|
|---|---|
|**Data structure**|How data is organized and stored|
|**Algorithm**|The steps used to process that data or solve a problem|

**Example:**

A **queue** stores customers in order.

The **algorithm** for helping them could be:

1. Add new customers to the back.
2. Take the customer from the front.
3. Help that customer.
4. Repeat until the queue is empty.

So:

> Data structures hold the information. Algorithms decide what to do with it.

**How this connects to previous units**

You already learned structures like:

- **Lists**
- **Dictionaries**
- **Stacks**
- **Queues**

This unit builds on those by focusing on the _plans_ used with them.

For example:

- A stack can support an **undo algorithm**.
- A queue can support a **first-come, first-served algorithm**.
- A list can be searched, sorted, updated, or processed with different algorithms.

**Why algorithms matter**

Learning algorithms helps you:

- Write cleaner code
- Solve harder problems
- Choose better steps for a task
- Improve software performance
- Think more clearly about how data gets processed

Your summary from the lesson was strong:

> Algorithms connect to data structures because algorithms dictate the logic for processing, while data structures define how that data is organized and stored.

And:

> Learning them helps optimize software performance and solve complex technical problems.

**Key takeaway**

> **Data structures organize data. Algorithms process data. Good programmers understand both.**
