## 📝 Notes
*   **Lesson: What is a data structure?**

**Core definition**

- A **data structure** = a _specific way of organizing data in memory_
- Goal: make storing and **retrieving** data **efficient**, not random or slow

**Why data structures matter**

- With small data, any messy approach “works”
- With huge data (thousands/millions), bad organization = slow and painful
- You always ask:
    - **How will I find items?**
    - **How will I add/remove items?**

**Lists vs Dictionaries (Python)**

- **List**
    
    - Like a row of lockers (positions: `0, 1, 2, ...`)
    - Access by **index/position** → `players[0]`
    - To find something by content (e.g., username), you often must **scan** the whole list
- **Dictionary**
    
    - Like labeled drawers
    - Access by **key/label** → `scores["Tylor"]`
    - Jumps directly to the value using the key (no full scan)

**Why different data structures exist**

- There is no “best” structure overall
- You pick the one that fits the job:
    - Need **order / positions** → list is a good fit
    - Need **lookups by label/key** → dict is a good fit
- We don’t use lists for everything because:
    - Sometimes we must **jump straight to data by a key**
    - Dicts handle that much more directly than lists
