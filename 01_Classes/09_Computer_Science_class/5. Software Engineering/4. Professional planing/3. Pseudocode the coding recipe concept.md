          - SHOW "Invalid username"
      - ELSE
          - SHOW "Username accepted"
      - END
* **Pseudocode** = plain, structured steps to plan logic before writing real Python.

## 📝 Notes
*  **1. What pseudocode is 🧠**

> Plain, structured steps that describe what the program should do, written in simple language.

Key points:

- It’s **not real Python**; the computer can’t run it.
- It’s for **planning and thinking**, not execution.
- It sits **between** the feature description and real code.

**2. Why it matters ⚙️**

We use pseudocode to:

- Turn **requirements** into **clear, ordered steps**.
- Catch missing cases early (like “what about age 18?”).
- Make the logic easy to read for you, teammates, and future-you.
- Make writing Python later faster and less confusing.

**3. How to format good pseudocode ✍️**

Patterns you practiced:

- Use a clear structure:
    
    - `START`
    - `GET ... from user`
    - `IF ...`
        - (indented action) `SHOW "message"`
    - `ELSE IF ...`
        - (indented action)
    - `ELSE`
        - (indented action)
    - `END`
- Style rules:
    
    - **One action per line.**
    - Use simple verbs: `GET`, `CHECK`, `SHOW`, `CALCULATE`.
    - Keep conditions specific:
        - “IF age is less than 13” (not “IF age is young”)
        - “IF code contains a space” (not “IF code is bad”)
    - Match messages to requirements exactly:
        - `"Invalid code"`, `"Password accepted"`, `"Adult user"`, etc.

**4. Examples you saw 👇**

- **Username rule** (3–10 chars, no spaces)
- **Password rule** (min 8 chars, no spaces)
- **Discount code** (exactly 6 chars, no spaces)
- **Age check** (<13, 13–17, ≥18)

For each one, the flow was:

> Feature description ➜ Pseudocode steps ➜ (later) Python code
