* **1. List of dicts**

> A **list of dicts** is a list where each item is a dictionary (a record).

Example:

```python
players = [
    {"name": "Alice", "role": "Tank"},
    {"name": "Bob",   "role": "Tank"},
    {"name": "Cara",  "role": "Healer"},
]
```

Use it when:

- You want **one big collection** of records.
- You often need to **loop through everything** (leaderboards, “show all players”, “show all movies”, etc.).
- It’s your **main storage** of data most of the time.

---

**2. Dict of lists**

> A **dict of lists** is a dict where each key is a _group name_ and the value is a list of items in that group.

Examples:

```python
players_by_role = {
    "Tank":   ["Alice", "Bob"],
    "Healer": ["Cara"]
}

words_by_first_letter = {
    "a": ["apple", "ant"],
    "b": ["banana", "boat"],
}
```

Use it when:

- You need to **group items by a key** (role, genre, first letter, category).
- You often ask: “**Given X, show me all items with X**” (given role, given genre, given category).
- You want **fast access** to whole groups:
    
    ```python
    players_by_role["Healer"]
    words_by_first_letter["b"]
    ```
    

---

**3. Group-as-you-go pattern (`dict.get + append`)**

Core pattern for building a dict-of-lists:

```python
groups = {}

for item in items:
    key = ...                     # how you group (role, first letter, etc.)
    current_list = groups.get(key, [])
    current_list.append(item)
    groups[key] = current_list
```

Key pieces:

- **`key`**: what you’re grouping by (e.g. `player["role"]`, `word[0]`).
- **`.get(key, [])`**:
    - If `key` exists → returns its list.
    - If `key` does NOT exist → returns `[]` (empty list) so you **don’t crash**.
- Then you `append` and assign back.

Concrete example (words by first letter):

```python
words = ["apple", "ant", "banana", "boat", "car"]
words_by_first_letter = {}

for word in words:
    first_letter = word[0]
    current_list = words_by_first_letter.get(first_letter, [])
    current_list.append(word)
    words_by_first_letter[first_letter] = current_list
```

---

**4. When to pick which**

- **List of dicts is better when**:
    
    - You want the **full list** of records.
    - You’re fine **looping through everything**.
    - You’re storing your main data.
- **Dict of lists is better when**:
    
    - You need **groups** (by role, genre, category, first letter, etc.).
    - You frequently ask: “**Given this key, give me all items in that group**”.
    - You want **instant jump** to a group using `my_dict[key]`.

Most real apps:

- Use **list of dicts** as main storage.
- Build **dict of lists** as a helper for fast grouped lookups.
