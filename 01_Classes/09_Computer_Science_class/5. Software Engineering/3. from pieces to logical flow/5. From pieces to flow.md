## 📝 Notes
* **Core idea**

- Build a **pipeline** of functions where:  
    **return value of one function → becomes the input of another**.

**Print vs return**

- `print(x)`
    - shows a value on the screen
    - does **not** give anything back to the caller
    - cannot be used as data for the next function
- `return x`
    - hands `x` back to whoever called the function
    - lets another function use that value

> To chain functions, you _must_ use `return`, not `print`.

**Basic pipeline pattern**

Python

```python
result_a = function_a(input_data)
result_b = function_b(result_a)
print(result_b)
```

Flow:  
`input_data → function_a → result_a → function_b → result_b`

**Your temperature example**

Python

```python
def change_to_fahrenheit(c):
    return c * 9/5 + 32

def add_offset_to_temp(f, offset):
    return f + offset

def main():
    c = 12
    f = change_to_fahrenheit(c)
    final_temp_with_offset = add_offset_to_temp(f, 2)
    print(final_temp_with_offset)

if __name__ == "__main__":
    main()
```

Data flow:  
`c (12) → change_to_fahrenheit → f → add_offset_to_temp → final_temp_with_offset`

Each helper **returns**, `main` does the **orchestration + printing**.

**List-of-dicts pipeline (tickets)**

Data:

Python

```python
tickets = [
    {"price": 10, "status": "valid"},
    {"price": 12, "status": "cancelled"},
    {"price": 8,  "status": "valid"},
]
```

Step 1 – filter:

Python

```python
def filter_valid_tickets(tickets):
    valid = []
    for ticket in tickets:
        if ticket['status'] == 'valid':
            valid.append(ticket)
    return valid
```

- Creates a new list `valid`
- Keeps only tickets where `status` is `"valid"`
- **Returns** that new list

Step 2 – summarize:

Python

```python
def calculate_total(tickets):
    total = 0
    for ticket in tickets:
        total += ticket['price']
    return total
```

- Starts `total` at `0`
- Adds each ticket’s `price`
- **Returns** the final sum

Orchestration in `main`:

Python

```python
def main():
    filtered = filter_valid_tickets(tickets)
    total_of_valid_tickets = calculate_total(filtered)
    print(total_of_valid_tickets)

if __name__ == "__main__":
    main()
```

Flow:  
`tickets → filter_valid_tickets → filtered → calculate_total → total_of_valid_tickets`

**Key habits you used (and should keep using)**

- Functions do **one job** (single responsibility).
- Helpers **do not print**, they only `return`.
- `main()`:
    - builds the input data
    - calls helper functions in order
    - stores intermediate results in clear variables (`filtered`, `total_of_valid_tickets`)
    - handles `print(...)`
