# 1.When a single dict isnt enough

*   When a single dictionary isnt enough
	* basically how to make tables in python

## 📝 Notes
* one row/record → dicitonary
*   a whole table is a list of dicitonaries

* **Core idea**

- Use a **list of dictionaries** when you have **many similar records** (table-like data).
- Think:
    - **Row / record** → one dictionary
    - **Table** → list of those dictionaries

**1. One record as a dictionary**

> A _record_ is one “thing” with several related fields.

Example – one friend:

Python

```python
friend = {
    "name": "Alex",
    "age": 25,
    "favorite_game": "Valorant"
}
```

- Keys = field names (like column headers): `"name"`, `"age"`, `"favorite_game"`.
- Values can be different types: `str`, `int`, etc.

**2. Many records as a list of dictionaries**

Turn a table like:

|name|age|favorite_game|
|---|---|---|
|Alex|25|Valorant|
|Josh|19|World of Warcraft|
|Samantha|25|Call of Duty|

into:

Python

```python
friends = [
    {"name": "Alex", "age": 25, "favorite_game": "Valorant"},
    {"name": "Josh", "age": 19, "favorite_game": "World of Warcraft"},
    {"name": "Samantha", "age": 25, "favorite_game": "Call of Duty"}
]
```

Pattern:

- Outer `[...]` → **list** = whole table
- Inner `{...}` → **dictionary** = one row / record
- All dictionaries share the **same keys**.

**3. Accessing data**

General pattern:

Python

```python
table[row_index]["column_name"]
```

Examples:

Python

```python
friends[1]["name"]           # second friend's name
friends[2]["favorite_game"]  # third friend's favorite game
```

Remember: Python lists are **0-based**:

- `friends[0]` → first row
- `friends[1]` → second row
- `friends[2]` → third row

**4. Another example: students**

Table:

|name|cohort|passed|
|---|---|---|
|Alex|1|True|
|Jamie|1|False|
|Sam|2|True|

Python:

Python

```python
students = [
    {"name": "Alex", "cohort": 1, "passed": True},
    {"name": "Jamie", "cohort": 1, "passed": False},
    {"name": "Sam", "cohort": 2, "passed": True}
]
```

Access:

Python

```python
students[1]["cohort"]   # Jamie's cohort
students[0]["passed"]   # Alex's passed status
```

**5. Why this structure is good**

Compared to `friend1`, `friend2`, … or three separate lists:

- **Organized**: All related data stays together per record.
- **Scalable**: 3 records or 10,000 records use the same pattern.
- **Programmable**: Easy to “do something for every record” (loops later).
- **Clear**: `students[2]["name"]` is self-explanatory.
