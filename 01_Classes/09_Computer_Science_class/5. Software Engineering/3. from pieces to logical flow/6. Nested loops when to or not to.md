# 6. Nested loops when to or not to

## 📌 Overview
*   **Course:** SE
*   **Lecture Date:** 2026-05-01 
*   **Instructor:** maestro 

## 🎯 Learning Objectives
*   when to or not to use nested loops

## 📝 Notes
*   **1. When nested loops are truly needed**

Use a **nested loop** when your data is naturally “thing → list of sub-things” and you must inspect _each_ sub-thing.

Examples:

Python

```python
players = [
    {"name": "Alice", "items": ["Sword", "Shield"]},
    {"name": "Bob", "items": ["Potion"]},
    {"name": "Cara", "items": ["Bow", "Sword"]},
]

for player in players:                 # outer
    for item in player["items"]:       # inner
        if item == "Sword":
            print(player["name"], "has Sword")
```

Here, you **must** scan each player’s items to _discover_ who has `"Sword"`. The nested loop matches the data shape.

Same with:

Python

```python
orders = [
    {"id": 1, "items": ["apple", "banana"]},
    {"id": 2, "items": ["banana"]},
    {"id": 3, "items": ["apple", "cherry"]},
]
# Need to know which orders include "apple" → nested loop is reasonable.
```

**2. Turning nested loops into dictionary lookups**

If you already have, or can build, a way to **jump directly** to what you need (by key), you can often remove a nested loop.

General pattern:

1. Build a **lookup dict** in one pass (maybe using a nested loop).
2. Later, use **one loop + dict lookup** instead of a nested loop.

Example: from `players` to `item → list of players`:

Python

```python
players = [
    {"name": "Alice", "items": ["Sword", "Shield"]},
    {"name": "Bob", "items": ["Potion"]},
    {"name": "Cara", "items": ["Bow", "Sword"]},
]

item_to_players = {}

for player in players:
    for item in player["items"]:
        names = item_to_players.get(item, [])
        names.append(player["name"])
        item_to_players[item] = names
```

Now you can do:

Python

```python
for name in item_to_players["Sword"]:
    print(name, "has Sword")
```

That print part is **one loop only** — minimal work.

**3. Id-based joins: classic place to avoid nested loops**

Data shape:

Python

```python
customers = [
    {"id": 1, "name": "Alice", "country": "US"},
    {"id": 2, "name": "Bob", "country": "CA"},
    {"id": 3, "name": "Cara", "country": "US"},
]

orders = [
    {"id": 101, "customer_id": 1, "total": 50},
    {"id": 102, "customer_id": 3, "total": 30},
    {"id": 103, "customer_id": 2, "total": 70},
]
```

Bad shape (unnecessary nested work):

Python

```python
for order in orders:
    for customer in customers:
        if customer["id"] == order["customer_id"]:
            print(customer["name"] + ":", order["total"])
```

Better: build a lookup once, then jump by key:

Python

```python
id_to_customer = {}
for customer in customers:
    id_to_customer[customer["id"]] = customer

for order in orders:
    customer = id_to_customer[order["customer_id"]]
    print(customer["name"] + ":", order["total"])
```

No nested loop needed in the second part.

**4. Quick decision checklist: keep nested loop or refactor?**

Ask yourself:

1. **Am I scanning a list just to find a match by some key (like `id` or `name`)?**
    
    - If yes, a dict lookup is probably better.
2. **Do I truly need to inspect _every_ combination of outer + inner elements?**
    
    - Example: every `player` with every `item` they own → nested loop is fine.
3. **Will I reuse this “find by key” logic many times?**
    
    - If yes, building a lookup dict once is even more worth it.

If:

- You’re doing a “join by id / name / key” → likely **refactor to dict**
- You’re walking a genuine hierarchy (object → children list) and must inspect each child → **nested is okay**
