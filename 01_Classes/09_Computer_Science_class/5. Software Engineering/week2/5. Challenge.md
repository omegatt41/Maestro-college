## 📝 Notes
* **1. Looping over dicts**

- `for key in d:`
    - Loops over **keys**.
    - Access value with `d[key]`.
- `for key, value in d.items():`
    - Loops over **(key, value)** pairs directly — cleaner when you need both.

**2. Filtering with `continue`**

Use `continue` to **skip** items that don’t match:

Python

```python
for letter, value in points.items():
    if value < 5:
        continue      # skip small values
    total += value    # only 5 or more
```

Pattern:

- “If this case is _bad_, `continue`.”
- The _good_ case is what’s left after the `if`.

**3. Safe lookup with `.get`**

Python

```python
scores = {"alice": 5, "bob": 8}
total = 0
for name in ["alice", "carol", "bob"]:
    total += scores.get(name, 0)
```

- `.get(key, default)` returns:
    - value if key exists
    - `default` (like `0`) if key is missing
- Great for **counting / summing** when some keys might not be there.

**4. Counting pattern with dicts**

Basic pattern:

Python

```python
counts = {}
for x in items:
    if x not in counts:
        counts[x] = 0
    counts[x] += 1
```

Used in the lesson with letters like `"a", "b", "a", ...`  
Key idea: **initialize once, then increment**.

**5. `break` in loops (esp. nested)**

- `break` stops the **current loop**, not all loops.

Python

```python
for key in data:
    for num in data[key]:
        if num == 4:
            break      # stop inner loop on this key
        total += num
```

- Outer loop (`for key in data`) keeps going.
- Inner loop (`for num in data[key]`) stops when hitting `4`.

**6. Updating inventory / scores**

Common mutation pattern:

Python

```python
if item in inventory and inventory[item] > 0:
    inventory[item] -= 1
```

- First check **key exists**.
- Then check **value > 0**.
- Then **update**.

This pattern shows up in:

- inventories
- hit points / health
- remaining seats, tickets, etc.
