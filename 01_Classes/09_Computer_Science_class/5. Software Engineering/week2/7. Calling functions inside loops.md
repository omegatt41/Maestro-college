# 7. Calling functions inside loops

## 📌 Overview
*   **Course:** SE
*   **Lecture Date:** 2026-04-25 
*   **Instructor:** Maestro 

## 🎯 Learning Objectives
*   Calling functions inside loops

## 📝 Notes
*   **1. Core pattern**

> Let the **loop** handle iteration and totals.  
> Let **helpers** handle the work for **one item** and return a value.

In code:

Python

```python
for item in items:
    result = helper(item)
    # use result (update totals, print, etc.)
```

**2. Helper functions (per-item work)**

- Take **one item** as an argument.
- Do the processing for that single item.
- Use `return`, not `print`, so the loop decides what to do.

Example:

Python

```python
def add_tax(amount):
    return amount * 1.1
```

**3. Loop responsibilities**

Inside the loop, the order should be:

1. Read the current item (`p`, `n`, etc.).
2. Call helper with that item.
3. Get the returned value.
4. Update accumulators / flags **after** the helper call.
5. Optionally print or store.

Your final code nailed it:

Python

```python
prices = [10, 5, 20]
total_with_tax = 0

def add_tax(amount):
    return amount * 1.1

def format_price_with_tax(amount):
    with_tax = add_tax(amount)
    formatted_price = f"Original price: ${amount}, with tax {with_tax}"
    return formatted_price

for p in prices:
    with_tax = add_tax(p)           # 2–3: helper call → result
    total_with_tax += with_tax      # 4: update accumulator AFTER helper
    formatted_final = format_price_with_tax(p)
    print(formatted_final)

print(f"Total: ${total_with_tax}")
```

**4. What NOT to do**

- Don’t put the whole loop inside the helper.
- Don’t update accumulators (like `total_with_tax`) inside helpers.
- Don’t rely on variables from outside the helper that you didn’t pass in.
- Don’t `print` from helpers when you actually need to _use_ the value later — `return` instead.

**5. Naming helpers**

Use clear **verb + noun** names:

- `add_tax(amount)`
- `format_price_with_tax(amount)`
- `evaluate_score(score)`
- `format_receipt_line(item)`

Name should describe: **“What does this do for ONE item?”**
