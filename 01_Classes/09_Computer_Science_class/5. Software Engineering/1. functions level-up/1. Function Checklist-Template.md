## 📝 Notes
*  When you write a function, ask yourself:

1. **What types do I expect?**
    
    - “Should this be `int`, `float`, or `str`?”
    - Do I need to guard against strings, `None`, etc.?
    - → If yes, add a **type guard** at the top.
2. **What values are not allowed?**
    
    - “Can this be zero?” (for division, percentages)
    - “Can this be negative?” (age, count, price)
    - “Is there a minimum or maximum?” (0–100 score, etc.)
    - → Add **value guards** like:
        - `if b == 0: return None`
        - `if age < 0: return None`
3. **What’s the “happy path”?**
    
    - After all guards pass, what is the _normal_ work?
    - Put that at the bottom as a clean `return`:
        
        ```python
        return a / b
        ```
        
4. **What should I return on bad input?**
    
    - `None` (like we did)
    - or maybe `0`, or a special value your app expects  
        The key: be **consistent**.
