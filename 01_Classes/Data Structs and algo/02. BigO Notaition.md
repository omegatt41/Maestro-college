# 02. BigO Notaition

## 📌 Overview
- **Definition**: To find the complexity of your Python code quickly:

	1. Drop constants: \(\mathcal{O}(2n + 5)\) simplifies cleanly to \(\mathcal{O}(n)\).
	2. Keep the dominant term: If a function has a single loop \(\mathcal{O}(n)\) followed by a nested loop \(\mathcal{O}(n^2)\), the total complexity is dictated by the worst factor: \(\mathcal{O}(n^2)\)

## 📊 Complexity Analysis
- **Time Complexity**: 
  - Best Case: $O(1)$ the item is the first item
  - Average Case: $O(n)$ the item you are looking for is at the middle
  - Worst Case: $O(n)$ the item your looking for is at the end
- **Space Complexity**: $O()$

## 💡 Key Insights & Patterns
**Big O analysis notes**

**Big O** describes how an algorithm’s work grows as input size `n` grows.

`n` = the size of the input  
Example: if a list has 100 items, `n = 100`.

**Main rule: keep the dominant term**

When writing Big O, focus on the part that grows fastest.

Examples:

Text

```text
3n + 20        → O(n)
5n² + 10n + 4  → O(n²)
n² + 10n + 50  → O(n²)
```

We drop:

- constant multipliers like `3n` → `n`
- smaller terms like `+ 10n`
- fixed numbers like `+ 50`

**Common code patterns**

Python

```python
print(items[0])
```

This is **O(1)** because it does one direct action.

Python

```python
for item in items:
    print(item)
```

This is **O(n)** because it loops once per item.

Python

```python
for a in items:
    for b in items:
        print(a, b)
```

This is **O(n²)** because it does `n * n` work.

**Important distinction**

`O(n²)` means quadratic:

Text

```text
n * n
```

`O(2^n)` means exponential:

Text

```text
2 doubled repeatedly as n grows
```

Nested loops over the same list are usually **O(n²)**, not `O(2^n)`.

**Quick memory hook**

Ask: “What part grows fastest when `n` gets huge?”

That answer is the Big O class.
